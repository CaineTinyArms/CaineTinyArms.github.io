Ludum Dare 55

bergs

This is a game created in 72 hours for Ludum Dare 55 based around the theme "summoning". Rapid iteration was the main focus on this project as the time constraint was so small.

Kotlin Android Studio Database Design API Integration
View GitHub Repository

Overview

Solve the puzzles using your power to summon additional bergs!

Co-developed for the Ludum Dare 55 Game Jam, bergs main focus was creating an fun prototype within the 72 time constraint. My main role for the creation of the game was the level-design, and programming small parts of the game.

This project was used to learn the Godot game engine, and the GDScript programming language.

Unlimited bergs!

Summon as many bergs as you desire, as long as there's room on the screen.

Work as a team!

The different berg sizes can be used to solve different puzzles.

Technical Breakdown

Technical implementation focused on usability, structure, and scalability.

The application was developed using Kotlin within Android Studio, with a focus on ease of use, maintainable architecture, responsive UI behaviour, and structured local data handling.

Additional development focused on interface clarity, information hierarchy, and creating an interaction flow that remained simple and readable across varying amounts of user data.

Reflection

Balancing technical implementation with usability-focused design.

One of the primary challenges during development was maintaining interface clarity while supporting increasingly complex data organisation and interaction flows.

The project became a valuable exploration of combining structured technical systems with accessible and visually readable user experience design.
